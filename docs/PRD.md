# PRD - 線上算命系統

## 1. 專案概述
- **背景與動機**：現代人生活壓力大、常面臨各種抉擇與迷惘，但往往沒時間或管道親自到實體廟宇或占卜館求神問卜。本系統旨在提供一個便捷的線上服務，讓使用者隨時隨地都能獲得指引與心靈慰藉。
- **目標用戶**：
  - 面臨職涯、感情、生活抉擇而感到迷惘，需要指點方向的人。
  - 對命理、占卜有興趣，尋求趣味性的年輕族群。
  - 喜歡傳統文化並尋求心靈寄託的一般大眾。
- **核心價值主張**：以具備儀式感的線上體驗設計，結合傳統算命與占卜，提供隨時可用、可留存且具回饋機制的個人化命理服務。

## 2. 功能需求

以下為系統核心五大主要功能與使用者故事：

1. **線上算命、占卜、抽籤**
   - 作為 **使用者**，我希望 **能透過線上介面進行抽籤、算命或占卜**，以便 **解答我目前的疑惑與生活迷惘**。
2. **會員個人系統**
   - 作為 **使用者**，我希望 **能註冊並登入帳號**，以便 **系統能辨識我的身份資訊並綁定後續操作**。
3. **儲存算命成果**
   - 作為 **使用者**，我希望 **能將每次算命或抽籤的結果自動儲存**，以便 **我日後隨時回顧與比較不同時期的占卜建議**。
4. **線上捐香油錢（奉獻回饋）**
   - 作為 **使用者**，我希望 **能在獲得指引後，透過系統進行捐香油錢的操作**，以便 **還願或表達謝意與支持**。
5. **社群分享結果**
   - 作為 **使用者**，我希望 **能一鍵將算命或籤詩結果分享到社群平台（如 IG、FB、LINE）**，以便 **與親友討論或單純分享生活趣味**。

## 3. 非功能需求
- **技術限制**：採用 Flask + Jinja2 作為後端與模板引擎，並使用 SQLite 作為輕量級關聯式資料庫。
- **效能與安全考量**：
  - 確保捐香油錢功能具備基本的交易資訊保護，訂單金額與紀錄不被輕易竄改。
  - 抽籤演算法應確保結果具備統計上的隨機性，避免結果一直重複。
  - 密碼儲存應具備加密保護（如雜湊處理）。

## 4. MVP 範圍
本專案的最小可行性產品（MVP）範圍定義如下：

- **Must Have**（核心必備）
  - 線上抽籤/占卜系統（包含求籤、抽籤、解籤的完整體驗）。
  - 基本會員註冊與登入。
  - 帳號內紀錄並儲存過去的算命歷史成果。
- **Should Have**（強烈建議）
  - 模擬捐香油錢功能（提供表單與訂單紀錄，可暫不真實串接金流 API）。
  - 響應式網頁設計（RWD），確保在手機瀏覽器上體驗良好。
- **Nice to Have**（加分項目）
  - 將算命結果以精美圖片格式分享至第三方社群媒體。
  - 每日運勢推播或首頁每日一籤功能。
  - 更進階複雜的算命模組（如塔羅牌陣、八字算命）。
